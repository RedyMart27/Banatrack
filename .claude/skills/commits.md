# Convención de Commits - BanaTrack

## Formato

## Tipos
- **feat**: nueva funcionalidad
- **fix**: corrección de bug
- **docs**: cambios en documentación
- **refactor**: reorganización de código sin cambiar funcionalidad
- **style**: cambios de formato, espacios, etc
- **test**: agregar o modificar tests
- **chore**: tareas de mantenimiento (dependencias, docker, etc)

## Ejemplos
- `feat: agregar modelo de lote`
- `fix: corregir cálculo de recobro`
- `docs: actualizar README con instrucciones de instalación`
- `refactor: separar lógica de calendario en servicio propio`
- `chore: agregar dependencias FastAPI al requirements.txt`

## Reglas
- Siempre en español
- Máximo 72 caracteres
- Presente, no pasado ("agregar" no "agregué")
- Un commit por tarea completada
- Hacer commit al final de cada sesión de trabajo
